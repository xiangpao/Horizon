# Horizon 本地启动说明

## 当前配置

- 本地目录：`D:\Horizon`
- AI 接口：SiliconFlow OpenAI-compatible API
- Horizon provider：`openai`
- 模型：`deepseek-ai/DeepSeek-V4-Flash`
- Base URL：`https://api.siliconflow.cn/v1`
- 输出语言：`zh`
- 密钥位置：`.env`，不要提交到 Git

## 一键运行

在 `D:\Horizon` 双击：

```powershell
run-horizon.cmd
```

或在 PowerShell 运行：

```powershell
cd D:\Horizon
.\run-horizon.cmd
```

默认流程：

1. 检查 `.env` 和 `data\config.json`
2. 自动修正 UTF-8 BOM，避免 JSON 解析失败
3. 执行 `uv sync --extra dev`
4. 运行 `uv run horizon --hours 24`
5. 生成日报到 `data\summaries\`
6. 同步 Jekyll 文章到 `docs\_posts\`
7. 保存运行日志到 `logs\`
8. 双击运行结束后停留在窗口，按任意键关闭

## 常用命令

只做依赖和配置检查，不运行日报：

```powershell
.\run-horizon.cmd -SetupOnly
```

运行测试后再生成日报：

```powershell
.\run-horizon.cmd -Test
```

抓取最近 48 小时：

```powershell
.\run-horizon.cmd -Hours 48
```

跳过依赖同步，直接运行：

```powershell
.\run-horizon.cmd -SkipSync
```

手动运行原生命令：

```powershell
$env:PYTHONUTF8="1"
uv run horizon --hours 24
```

更新到最新版：

```powershell
git pull --ff-only
uv sync --extra dev
```

运行测试：

```powershell
uv run pytest
```

## 输出文件

- 本地日报：`data\summaries\horizon-YYYY-MM-DD-zh.md`
- GitHub Pages 文章：`docs\_posts\YYYY-MM-DD-summary-zh.md`
- 运行日志：`logs\horizon-YYYYMMDD-HHMMSS.log`

直接查看最新日报：

```powershell
notepad data\summaries\horizon-2026-06-30-zh.md
```

如果当天日期不同，把文件名里的日期换成当天日期。

## GitHub Pages 和邮箱怎么看

- 本地生成的 `docs\_posts\*.md` 只是站点文章源文件，不会自动发布到线上。
- 你的 GitHub Pages 目标站点：`https://xiangpao.github.io/Horizon/`。
- 当前本地仓库上游是 `Thysrael/Horizon`，没有推送权限；需要使用 `xiangpao/Horizon` fork 后才能发布。
- 自动日报需要在 GitHub 仓库 Secrets 里配置 `SILICONFLOW_API_KEY`，否则 GitHub Actions 无法调用模型。
- 邮箱当前是关闭的：`data\config.json` 里 `email.enabled=false`，所以不会收到邮件。
- 要启用邮箱，需要配置 SMTP/IMAP、发件邮箱授权码、订阅者列表，再把 `email.enabled` 改成 `true`。

## 操作注意事项

- `.env`、`data\config.json`、`data\summaries\*.md`、`docs\_posts\*.md` 已被 `.gitignore` 忽略，本机运行不会默认提交密钥或生成日报。
- `环境配置.txt` 里有 API key，目前是未跟踪文件；不要提交、截图或发给别人。
- PowerShell 写 JSON 容易带 BOM；如果手改配置后启动失败，优先运行 `.\run-horizon.cmd -SetupOnly` 修正编码。
- 当前模型选择 `DeepSeek-V4-Flash`，适合批量打分和日报生成；需要更高质量时，把 `data\config.json` 里的 `model` 改成 `deepseek-ai/DeepSeek-V4-Pro`。
- 如果 API 报 401，检查 `.env` 的 `OPENAI_API_KEY`；如果报模型不存在，检查 SiliconFlow 控制台里模型名称是否可用。
- 如果 Docker 报 daemon 连接失败，先启动 Docker Desktop；本地运行不依赖 Docker。
- 发布到线上 GitHub Pages 需要走仓库 workflow 或手动提交 `docs` 内容；本地生成只说明文件已产出，不等于已发布到线上。
